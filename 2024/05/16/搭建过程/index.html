搭建过程

工作流

  • 写博客
    • hexo -n "${name}"
    • 写
    • hexo s -g本地预览
    • hexo d -g部署
  • 写awesome-papers：win下运行deploy.bat，push后利用actions自动部署

搭建过程

  • npm install -g hexo-cli

  • 初始化项目：在D:/blog

    此时tree：

  • hexo s (server)即可本地运行

  • 创建andakai.github.io仓库

  • 配置github pages：在仓库的settings中设置即可

  • 上传到github：在_config.yml中改

  • hexo d即可部署了，就可以访问andakai.github.io了

在这个对应目录的polyfills.js的第65行起，注释3行

之后仍报错：

参考：安装gitbook出现下列问题，求解_error: failed to parse json unexpected token ‘u’ a-CSDN博客
使用nvm进行管理，降低node的版本至12.22.12
重新安装gitbook-cli，并修改polyfills.js。
安装成功。
2. 部署

在生成的SUMMARY.md的修改目录，类似：

gitbook serve (--port 2333)本地部署
3. 安装插件：在根目录创建book.json，并运行gitbook install

  • 创建一个仓库awesome-papers
